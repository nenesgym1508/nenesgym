# 📋 REGISTROS — Documentación de avance del proyecto

Esta carpeta es la **bitácora oficial** de NENE'S GYM. Sirve para no perder el contexto de qué se
ha hecho, por qué, y cómo se resolvieron los problemas. Se versiona junto al código (git).

## Archivos

| Archivo | Para qué sirve |
|---|---|
| [`AVANCE.md`](./AVANCE.md) | Progreso del proyecto **por secciones/features**: qué está hecho, en proceso o pendiente. |
| [`ERRORES.md`](./ERRORES.md) | **Errores detectados** y cómo se solucionaron (síntoma → causa → solución). |

## Regla de oro

> Cada vez que se **completa una sección** o se **resuelve un error**, se registra aquí en el mismo
> momento. La documentación se actualiza junto con el cambio, no después.

## Cómo registrar un avance (en `AVANCE.md`)

1. Actualiza el estado en la **tabla resumen** (✅ Completado · 🚧 En progreso · ⏳ Pendiente).
2. Agrega o actualiza la sección de detalle con una línea de qué se hizo y la fecha.
3. Si algo quedó pendiente, muévelo a la lista de **Próximos pasos**.

## Cómo registrar un error (en `ERRORES.md`)

1. Asigna el siguiente ID correlativo (`ERR-009`, `ERR-010`, …).
2. Añádelo a la **tabla resumen** con fecha y estado.
3. Documenta el detalle: **Síntoma**, **Causa raíz**, **Solución**, **Estado**.

## Convenciones

- **Idioma:** español.
- **Fechas:** formato `AAAA-MM-DD`.
- **Estados:** ✅ Resuelto/Completado · 🚧 En progreso · ⏳ Pendiente · ⚠️ Requiere acción.
